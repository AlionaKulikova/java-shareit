# ShareIt
ShareIt — это сервис для шеринга вещей, позволяющий пользователям делиться своими предметами и брать их в аренду на определенное время. Проект разработан с использованием Spring Boot и включает микросервисную архитектуру для обеспечения масштабируемости и производительности.

## Описание проекта
В мире, где экономика совместного использования набирает популярность, ShareIt помогает пользователям эффективно делиться вещами. Представьте: вам нужна дрель для ремонта, но покупать её бессмысленно. С ShareIt вы можете одолжить её у друга или другого пользователя сервиса, забронировать на нужные даты и даже оставить отзыв после использования.

## Основные сущности
Item: Вещь с названием, описанием и статусом доступности. Владелец — пользователь, добавивший вещь.
User: Пользователь сервиса.
Booking: Бронирование вещи на определенные даты (будет добавлено позже).
ItemRequest: Запрос на вещь, если она не найдена (будет добавлено позже).

## Эндпоинты для Item:
POST /items — добавить новую вещь (требуется X-Sharer-User-Id в заголовке).
PATCH /items/{itemId} — редактировать вещь (только владелец).
GET /items/{itemId} — просмотреть вещь.
GET /items — список вещей пользователя.
GET /items/search?text={text} — поиск доступных вещей по тексту.
POST /items/{itemId}/comment — добавить отзыв (только после аренды). 
GET /items/{itemId} и GET /items.

## Эндпоинты для Booking:
POST /bookings — создать запрос на бронирование (статус WAITING).
PATCH /bookings/{bookingId}?approved={approved} — подтвердить/отклонить (только владелец; статус APPROVED/REJECTED).
GET /bookings/{bookingId} — данные о бронировании (автор или владелец).
GET /bookings?state={state} — список бронирований пользователя (состояния: ALL, CURRENT, PAST, FUTURE, WAITING, REJECTED).
GET /bookings/owner?state={state} — бронирования вещей пользователя.

## Эндпоинты для Requests:
POST /requests — создать запрос вещи.
GET /requests — список своих запросов с ответами.
GET /requests/all — запросы других пользователей.
GET /requests/{requestId} — данные о запросе.
POST /requests — создать запрос вещи.
GET /requests — список своих запросов с ответами.
GET /requests/all — запросы других пользователей.
GET /requests/{requestId} — данные о запросе.

## Тестирование
Используйте Postman-коллекцию для проверки API.

## Микросервисная архитектура
Разделение на два приложения:
shareIt-server: Основная логика (порт 9090).
shareIt-gateway: Валидация и контроллеры для пользователей (порт 8080).
Многомодульный проект: Один репозиторий с двумя модулями.
Общение: Через REST.

### Запуск
shareIt-gateway (порт 8080) и shareIt-server (порт 9090).

## Использование
Регистрируйте пользователей через API.
Добавляйте вещи и бронируйте их.
Оставляйте отзывы после использования.
